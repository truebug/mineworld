## 1. 自上而下：产品 → 空间

```text

        │ 进入大厅
        ▼
┌────────────────── Hangar Core（本仓 Godot Hub）──────────────────┐
│  中央广场 · 电梯 → L2 观景廊                                       │
│                                                                   │
│  东翼 TYPE A 本仓关卡     北翼 TYPE B 卡片通道     西翼 TYPE C 边缘 │
│  A 工坊 · B 训练场        C 设计 · 展柜/教室        D 任务/真机坞   │
│                                                                   │
│  南翼 · E 竞技场门 + 外延甲板（未来船坞 / 建筑群）                    │
└───────────────────────────────┬───────────────────────────────────┘
                                │ 视觉外场（网格甲板 + 星空）
                                ▼
                     后续：停机坪、模块舱、接驳廊（不进本期权威）
```

| 类型 | 空间落点 | 门 / 交互 | 实现原则 |
|------|----------|-----------|----------|
| **A · 本仓关卡** | **东翼**为主；训练场仍可走西/南过渡 | A Workshop · B Training · E Arena | `join.level_id` → Gateway MuJoCo |
| **B · PMS 卡片** | **北翼**走廊 | C Design stub · Gallery/Classroom · 展柜 F | 元数据 URL；不进 Hub MjData |
| **C · 边缘真机** | **西偏北坞口** | D Missions / Edge Dock stub | 叙事 + 未来 edge enter；本期不透传真机 |

铁律不变：Hub 不录 IL；外部通道不塞 MJCF。
