# ADR-002：物理权威与时序同步

| 字段 | 值 |
|------|-----|
| **状态** | Proposed |
| **日期** | 2026-07-17 |
| **决策编号** | ADR-MW-002 |

---

## 背景

客户端与仿真服务分离后，必须明确「谁说了算」以及时间轴如何对齐，否则回放、训练与多人观战均不可信。

## 决策

1. **机甲及标注为 `mujoco_authoritative` 的实体**：位姿与关节状态以 MuJoCo 为准。
2. **仿真 `tick`** 为全系统时间 SSOT；`t_sim = tick * dt`。
3. **GDevelop** 对权威实体仅做插值/外推展示，不在本地积分物理。
4. **cmd** 带客户端 `tick` 或时间戳，Gateway 按到达顺序并入当前或下一仿真步（具体队列策略在 Gateway 实现中文档化）。
5. **录制** 以 Gateway 侧合并后的 cmd + state 为准，不采信客户端自报状态。

## 控制模式（MVP）

默认采用 **`velocity` 高层控制**，降低关节带宽与玩家操作门槛；关节级录制可作为 P1 扩展。

## 后果

| 正面 | 代价 |
|------|------|
| 训练数据物理一致 | 网络延迟影响操控手感，需预测/缓冲 |
| 回放可复现 | 客户端表现略滞后于仿真 |
| 反作弊：状态不由客户端上报 | Gateway 算力与带宽集中 |
