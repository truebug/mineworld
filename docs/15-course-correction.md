## 1. 最初设想（冻结意图）

一句话（与 [00-vision](00-vision.md) 一致）：

**用娱乐壳规模化采集「真物理人类遥操」数据**——不是做游戏，也不是做仿真器单体，更不是做自动驾驶。

| 层 | 意图 |
|----|------|
| **娱乐壳** | Godot 关卡 / 任务 / 反馈，降低获客与标注成本 |
| **真物理权威** | 无头 MuJoCo：关节、接触、可复现动力学 |
| **遥操档案** | `cmd` + `state`(+`joints`) + 任务事件落盘，可回放、可训、可评测 |
| **飞轮** | 好玩 → 更多人玩 → 更多有任务约束的轨迹 → 更好策略 / 更好卖数据 |

明确 **不是**：

- 自动驾驶 / 车道保持 / 规划 benchmark
- 用 Godot 物理硬扛机器人动力学
- 元宇宙社交或全感官头号玩家复刻

## 2. 实际走完的路径（时间线摘要）

按仓库已落地能力粗排（细节见 [09](09-todo.md) · [12](12-status-review.md) · [13](13-web-multiplayer-demo.md)）：

```text
契约 / Schema / 假物理网关
    → MuJoCo 权威 + 静态障碍 + 录制
    → Godot 傀儡跟 state（GDevelop → Godot）
    → Web 导出 + 本机 COOP/COEP Demo
    → Room / demo 房两人 + joints 出口
    → F0–F8 融合（DiffBot URDF、差速轮、互撞、跟皮）
    → demo_city 演示打磨（HUD、通关、KayKit、空气墙、seed、沥青路、小品）
    → 录制列表 / 2D·3D 回放 / SQLite 索引 / CSV 导出
```

**管道侧结果：成功。**  
「进关 → 遥操 → MuJoCo 驱动 → 落盘 → 回放/导出」闭环已通，铁律（Godot 不权威、MuJoCo 不叙事、Gateway 边界）未破。

## 3. 跑偏在哪里（诊断）

### 3.1 表象

玩家体感与数据语义都像：

> **遥控平面小车 / 履带车在街区开到绿点**（`velocity(vx, vy, ω)` + planar DiffBot）

而不是：

> **机甲关节级遥操 + 丰富接触任务 → 高信息密度人类示范**

### 3.2 根因（不是「做错了引擎」）

| 维度 | 设计目标 | POC 现状 | 偏差 |
|------|----------|----------|------|
| 控制接口 | 可升到 `joint_targets` / 多 DoF | 长期停在平面 `velocity` | **控制深度不足** |
| 机体 | 可换真机 MJCF/URDF | DiffBot planar 包装为主 | **DoF 叙事偏「车」** |
| 任务 | 有接触、失败、纠偏 | 到点通关 + 轻量推箱 | **任务过浅** |
| 世界皮 | viewer_only 装饰够用即可 | 大量精力在城市观感 / seed / 路面 | **演示打磨超前于数据价值** |
| 数据管道 | 与娱乐同等优先级 | 录制/回放/导出已较完整 | **管道超前、内容落后** |

### 3.3 容易误判的「假偏航」

| 误判 | 澄清 |
|------|------|
| 「这是在做自动驾驶」 | 没有感知栈、没有规划 benchmark；偏的是 **浅控制的车感**，不是 AD 产品 |
| 「Web / 多人是跑偏」 | 获客与 e2e 验收需要壳；偏的是 **壳打磨排到了控制深度前面** |
| 「MuJoCo 没用上」 | 权威链路、障碍、推箱、互撞、joints 均已验证；缺的是 **更有价值的控制与任务** |
| 「整条主线错了」 | 双引擎拆分与录制 SSOT 仍然正确；应 **纠偏优先级，而不是推翻架构** |

### 3.4 一句话定责

> **架构与管道正确；控制语义与任务深度滞后于演示皮。**  
> 跑偏 = 优先级跑偏，不是方向性背叛愿景。

## 4. 纠偏计划（讨论后写入 09 再开干）

原则：**先加深控制与任务，再扩关卡皮与公网。**  
公网 HTTPS / T2.7 手感仍可暂缓。

### 4.1 纠偏主题（建议 ID 前缀 `V` = Value）

| ID | 主题 | 目标 | 验收方向（草案） |
|----|------|------|------------------|
| **V1** | 控制升维 | 引入 `joint_targets`（或轮臂混合）并与 state/`joints` 成对录制 | 协议样例 + smoke；回放可见关节角变化 |
| **V2** | 机体升维 | 非 planar 多关节 mech（或手臂附加）进 `demo` 主线之一 | headless + mujoco smoke；Godot 跟皮 |
| **V3** | 接触任务 | 推箱升级：对准 / 堆叠 / 门或抓取点；失败可再接管 | 任务事件进 header；成功率可统计 |
| **V4** | 数据分层 | outcome + 子目标 + 难度标签；导出按任务过滤 | CSV/JSONL 含标签列；index.sqlite 可查 |
| **V5** | 演示克制 | 城市皮只做必要修补；不再开新地图包专题 | `09` 中 D\* 观感项默认 `[~]` |

### 4.2 明确不做（纠偏期）

- 自动驾驶式车道/规划评测集
- 公网运维优先于 V1–V3
- 在 Godot 内嵌物理权威或客户端落盘
- 未选清许可证前批量「任意 URDF」

### 4.3 与已完成工作的关系

| 已完成 | 纠偏中如何用 |
|--------|----------------|
| T2.6 `joints` / `joint_vels` | V1 的出口已铺好 |
| F2–F8 URDF/跟皮/差速 | V2 的工程钩子 |
| D2 推箱 | V3 的最小接触基线 |
| D5/D8/D13 录制回放导出 | V4 的管道基线；提高的是 **内容标签与控制深度** |
| demo_city 空气墙 + seed | 保留为娱乐壳；不再作为主投入面 |

### 4.4 决策闸门

后续具体勾选 **等本轮文档入库后的 todo 讨论** 再冻结进 [09-todo](09-todo.md) 的 `Now（V）` 表。  
本文只固定 **诊断与优先级方向**，避免讨论前又开新的城市皮专题。
